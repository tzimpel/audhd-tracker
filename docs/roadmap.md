# Roadmap

## Delivery Principle

V1 should be delivered in small, useful increments. Each increment should leave the app in a runnable and coherent state.

## Phase 0: Foundation

Outcome:

- agreed scope
- agreed stack
- documentation baseline

Deliverables:

- README
- product concept
- architecture concept
- implementation backlog
- agent standards
- contributor workflow conventions

## Phase 1: App Shell And Domain Foundation

Outcome:

- runnable Ionic PWA shell
- core folder structure
- domain types and repository interfaces

Deliverables:

- Ionic project scaffold
- routing shell
- theme tokens
- domain entities
- repository contracts
- stub local repository
- GitHub Actions CI workflow
- GitHub Pages deployment workflow
- initial testing setup

## Phase 2: Onboarding And Settings

Outcome:

- user can configure personal warning signs and basic preferences

Deliverables:

- onboarding flow
- warning sign management
- local settings persistence
- reduced-stimulus mode setting

## Phase 3: Daily Check-In

Outcome:

- user can log core daily state quickly

Deliverables:

- check-in screen
- capacity input
- main strain input
- rest helping input
- warning sign quick selection
- local save flow

## Phase 4: Home Summary

Outcome:

- user sees current state and concise explanation

Deliverables:

- home screen
- recent summary logic
- plain-language classification
- one recommended recovery action

## Phase 5: Trends

Outcome:

- user can see short-term patterns

Deliverables:

- 7-day trend view
- repeated strain indicator
- repeated warning sign indicator
- recent restoration pattern

## Phase 6: Recovery And Support

Outcome:

- app remains useful in low-capacity states

Deliverables:

- recovery plan screen
- minimum viable day template
- support mode screen/state
- basic calming/support actions

## Phase 7: Weekly Reflection

Outcome:

- user can review weekly strain and support patterns

Deliverables:

- weekly reflection form
- weekly history list
- lightweight summary

## Phase 8: Export / Import

Outcome:

- user can back up or restore their local data

Deliverables:

- JSON export
- JSON import
- schema validation
- version metadata

## Phase 9: Hardening

Outcome:

- app is stable enough for ongoing personal use

Deliverables:

- domain tests
- storage tests
- empty states
- error states
- install/offline validation
- UX polish
- branch protection / merge quality checklist if useful for the repository

## Deferred Beyond V1

- backend sync
- Firebase integration
- push notifications
- richer adaptive questioning
- calendar and passive-data integrations
- native wrapper or app-store deployment
